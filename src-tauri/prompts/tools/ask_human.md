<summary>主动挂起并请求人类协助。核心参数: message(求助信息)。</summary>

当你遇到无法自动处理的障碍（例如：必须手动拖动的滑块验证码、必须用手机微信扫描的登录二维码、必须由用户做出的艰难业务决策等）时，请调用此工具。

### 工具参数 (Params)
- `message` (String, 必填): 你对人类的求助信息。必须准确描述当前的情况以及你需要人类做什么。例如："页面出现了一个滑块验证码，我无法处理，请在浏览器中手动完成验证后告诉我。" 或 "需要扫码登录，请扫描页面上的二维码，完成后告诉我。"

### 执行逻辑
调用此工具后，你的所有动作将**彻底停摆并进入休眠状态**。前端会弹出你的 `message` 并等待人类操作。
直到人类完成操作并在前端输入结果后，你才会被唤醒。人类回复的文本将作为此工具的 `stdout` 结果返回给你。

### 使用范例

```json
{
  "reflection": "页面被反爬机制拦截，弹出了一个非常复杂的点选验证码。我作为AI无法完成这种需要人类视觉和交互的操作。",
  "thought": "我必须立刻挂起当前流程，把控制权交给人类老板，让人类帮我过验证码。完成之后我再继续抓取数据。",
  "todo_update": [],
  "tool": "ask_human",
  "commands": [
    {
      "action": "ask_human",
      "message": "老板，页面弹出了一个点选验证码，我过不去。请你在浏览器里手动点一下，完成验证之后告诉我一声！"
    }
  ]
}
```

### 注意事项
1. 这是一把**极其昂贵的钥匙**。不到万不得已，不要打扰人类。如果你可以通过 `click` 或 `type` 尝试绕过错误，应当优先尝试自动化解决。
2. 只有在**确信自动化无法继续**的死胡同时，才能使用 `ask_human`。
